# 実装仕様: visa-mcp v2.6.0 — export shim を lab-executor v2.18/v2.19 に同期

> **この仕様は Codex に実装を委ねるための自己完結ドキュメントです。**
> `tests/test_v2_6_export_shim_sync.py` が全 PASS になれば完了です。
> **修正が必要になった場合も Codex 側で対応してください。**

## 1. 背景 / 目的

visa-mcp の MCP server (`visa-mcp serve`) は `visa_mcp.tools.export` の
**独自コピー**を登録する (lab-executor とは別ファイル)。過去に
v2.13.2 の rows=0 修正が visa-mcp 側に届かず再発した経緯があり
(`test_v2_1_2_results_integration.py` 参照)、export 機能は両リポジトリで
同期し続ける必要がある。

lab-executor 側で以下が入った:
- **v2.18.0**: `RESULT_COLUMNS` に `sweep_index` / `sweep_value` 追加。
  `VISA_MCP_EXPORT_DIR` で export 先上書き (`_resolve_export_dir`)。
  export dir 作成失敗を structured error 化。
- **v2.19.0**: `get_experiment_results` / `export_experiment_results` に
  `instrument` / `sweep_index` / `measurement` フィルタ引数
  (`_filter_rows` 純ヘルパ)。

visa-mcp の `tools/export.py` はこれらが未反映 (pre-v2.18 状態)。
**runtime の JobManager は `lab_executor.job.JobManager` の re-export**
(`visa_mcp.job` は shim) なので、instrument / sweep の step result 注入は
既に上流で行われている。よって visa-mcp 側は **export.py の列・フィルタ
ロジックを揃えるだけ**で runtime でも列が埋まる。

## 2. 同期する機能 (lab-executor v2.18 + v2.19 と同一挙動)

すべて `src/visa_mcp/tools/export.py` 内。lab-executor の同名実装を
そのまま移植する。**visa-mcp 固有の差分は保持すること** (下記 §4)。

### 2.1 `RESULT_COLUMNS` に sweep 列追加 (v2.18)

既存タプル末尾に追加 (既存列の順序・名前は不変):

```python
RESULT_COLUMNS = (
    "timestamp", "target_id", "instrument", "measurement",
    "value", "unit", "step_index", "step_path",
    "sweep_index", "sweep_value",   # v2.6 同期 (lab-executor v2.18)
)
```

### 2.2 `_resolve_export_dir()` 新設 + `_safe_export_path` 改修 (v2.18)

```python
def _resolve_export_dir() -> Path:
    raw = os.environ.get("VISA_MCP_EXPORT_DIR", "").strip()
    if raw:
        return Path(raw).expanduser()
    return DEFAULT_EXPORT_DIR
```

`_safe_export_path` 内:
```python
base = _resolve_export_dir().resolve()
try:
    base.mkdir(parents=True, exist_ok=True)
except OSError as e:
    return (None, {
        "error_class": "export_dir_not_writable",
        "message": (
            f"export ディレクトリを作成できません: {base} ({e}). "
            "VISA_MCP_EXPORT_DIR で書き込み可能な場所を指定してください。"),
        "recoverable": True,
        "details": {"export_dir": str(base)},
        "recommended_next_actions": [
            {"action": "set_export_dir_env",
             "reason": "VISA_MCP_EXPORT_DIR=<書き込み可能 path> を設定"},
        ],
    })
```

- env 未指定時は `DEFAULT_EXPORT_DIR` 定数を返す (既存 monkeypatch 互換)。
- 「output_path は base 配下」既存 security check は維持。

### 2.3 `_extract_result_rows` で sweep 値を載せる (v2.18)

`common` dict に追加 (instrument は既存どおり `r.get("instrument")`):
```python
"sweep_index": r.get("sweep_index"),
"sweep_value": r.get("sweep_value"),
```
target_runs / monitor_data 由来の row にも `"sweep_index": None`,
`"sweep_value": None` を入れて CSV fieldnames と整合させる。

### 2.4 `_filter_rows` 純ヘルパ + フィルタ引数 (v2.19)

```python
def _filter_rows(rows, *, instrument=None, sweep_index=None,
                 measurement=None) -> list[dict]:
    out = rows
    if instrument:
        out = [r for r in out if r.get("instrument") == instrument]
    if sweep_index is not None:
        out = [r for r in out if r.get("sweep_index") == sweep_index]
    if measurement:
        out = [r for r in out if r.get("measurement") == measurement]
    return out
```

`get_experiment_results` / `export_experiment_results` の **両方**に
optional 引数 `instrument: str = ""`, `sweep_index: int | None = None`,
`measurement: str = ""` を末尾追加。`_extract_result_rows` 直後・
pagination 前に `_filter_rows` を適用 (pagination / rows 件数はフィルタ後)。
両ツールの response data に `filters` を echo:
```python
"filters": {"instrument": instrument or None,
            "sweep_index": sweep_index,
            "measurement": measurement or None},
```

## 3. version / sentinel

- `visa_mcp.__version__` を `2.6.0` に (`src/visa_mcp/__init__.py` +
  `pyproject.toml`)。
- export.py の `_meta.versions.export_fix` sentinel を **`"v2.6.0"`** に
  更新 (現状 `"v2.1.3"`)。これは「この visa-mcp の export 経路が
  sweep 列 + フィルタ対応済み」を client が判定するための印。

## 4. 保持する visa-mcp 固有差分 (lab-executor と意図的に違う箇所)

以下は **lab-executor からコピーして上書きしないこと**:

1. `_meta.versions` ブロックは visa_mcp と lab_executor の **両方**の
   version を載せる既存構造を維持 (lab-executor 版は lab_executor のみ)。
   `export_fix` だけ §3 のとおり更新。
2. `export_experiment_bundle` の manifest は `visa_mcp_version` キーを
   使う既存構造を維持。
3. コメント中の visa-mcp 版番号参照 (例 v2.1.2 / v2.2.1) はそのままでよい。
